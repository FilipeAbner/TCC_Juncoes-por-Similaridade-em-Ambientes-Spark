Significado dos Hearders de arquivo:

as 6 primeiras linhas indicam
1046  -> quantidade de tuplas
1 -> se o objeto tem ID (ou seja, a primeira coluna não é característica)
9 -> número de dimensões
1 -> Se o objeto tem alguma informação ao final, i.e., texto (ou seja, a última coluna não é característica)
2048   -> Era o tamanho do bloco usado na estrutura de arquivo, se não tiver implementado blocagem pode excluir a linha. 
autos  -> nome do dataset, caso tenha algum código na main que referencie arquivo (pode ser retirado também se não foi implementado). 

Logicamente, para realizar os experimentos você deve criar conjuntos de consulta com k elementos (10 ou 100)  escolhidos aleatoriamente do dataset.
